# 代码审查技能

## 何时使用此技能
- 用户提交了 Java 代码需要审查
- 需要检查代码质量、安全性、性能问题
- 需要生成代码审查报告

## 审查维度
1. **代码规范**：命名规范、注释完整性、代码格式
2. **性能优化**：算法效率、资源使用、数据库查询优化
3. **安全性**：SQL 注入、XSS 防护、敏感信息处理
4. **可维护性**：代码结构、设计模式、耦合度
5. **异常处理**：异常捕获、日志记录、错误信息

## 使用流程
1. 读取用户提供的 Java 代码
2. 逐维度分析代码质量
3. 标注问题等级：🔴严重 / 🟡警告 / 🔵建议
4. 给出具体的修改建议和示例代码
5. 生成审查报告

## 输出格式
```
【代码审查报告】

审查文件：{文件名}
审查时间：{时间}
总体评分：{分数}/10

## 发现问题

### 🔴 严重问题
- {问题描述}
  位置：第{行号}行
  建议：{修改建议}

### 🟡 警告
- {问题描述}
  位置：第{行号}行
  建议：{修改建议}

### 🔵 建议
- {问题描述}
  位置：第{行号}行
  建议：{修改建议}

## 总结
{总体评价和改进方向}
```
