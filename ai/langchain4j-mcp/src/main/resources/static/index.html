🔧 LangChain4j MCP Tutorial

Model Context Protocol — 三种传输方式 + 多 Provider + AI Service

1. stdio 传输

STDIO

通过标准输入/输出与本地 MCP Server 子进程通信。最简单的本地集成方式。

POST /api/stdio/ask?question=

2. HTTP/SSE 传输

LEGACY

通过 HTTP 请求和 SSE 通道与远程 MCP Server 通信（已弃用，建议使用 Streamable HTTP）。

POST /api/http/ask?question=

3. Streamable HTTP 传输

RECOMMENDED

MCP 2025-06-18 规范标准传输方式，支持 SSE 流和常规 HTTP 响应。

POST /api/streamable/ask?question=

4. 多 MCP Provider

MULTI

组合多个 MCP Client，让 LLM 同时使用来自不同 Server 的工具。

POST /api/multiprovider/ask?question=

5. AI Service + MCP

AI SERVICE

标准的 AI Service 模式：ChatModel + McpToolProvider → AiServices 组装。

POST /api/weather/ask?question=

系统状态

HEALTH

检查应用和 MCP 连接状态

GET /api/health
